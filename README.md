## 二、详细过程说明 —— 毕设推进

**背景**：毕设项目为针对历史人物评价的生成和识别研究，需要从《二十四史》等古文典籍中自动识别包含人物评价的句子，提取主体人物及评价信息。但是，当前系统在NER模型效果、大模型调用成本、智能问答能力等方面存在多个技术瓶颈。

### 2.1 系统架构重构

A. **问题复现**：原有系统采用多层级规则投票机制识别主体，包括基于正则的粗筛、词表匹配等层级。该方案存在规则冲突、对新文本泛化能力差等问题，识别效果不理想。

B. **导师指导**：建议使用微调后的开源模型替代多层级规则，将主体识别从"规则驱动"转向"模型驱动"。

C. **重构实现**：

（1）架构调整：移除原有的多层级投票逻辑，改为"微调NER模型主导 + 轻量后处理过滤"的单层架构。微调模型作为唯一主识别器，后处理仅做黑名单过滤和实体长度验证。

（2）模型选型：选用GujiRoBERTa作为基座模型，因其在中文古文NER任务上表现优于通用BERT。

（3）数据准备：按照导师指导，先对全文进行NER主体识别提取段落评价，再用古文模型筛选人物评价段落，将句子级数据压缩为段落级数据集，降低后续处理量。

（4）微调方法：采用LoRA（低秩适应）微调技术。相比于全参数微调，LoRA只训练模型中极小一部分参数，显存占用更低，训练速度更快，同时效果接近全量微调。

（5）置信度机制设计：实体级置信度采用token最小概率策略。原因是一个实体中任何一个token识别错误会导致整个实体错误，用token概率的最小值比平均值更合理。句子级置信度取该句子中所有实体的最高置信度，用于判断该句子是否值得保留。

（6）实体验证过滤：长度限制为2至6个中文字符；黑名单过滤排除虚词组合及常见误判词；字符类型限制仅保留纯中文；排除以特定虚词结尾的短词。

### 2.2 实体规范化

A. **设计原则**：微调模型作为主识别器，输出原始实体名称。CBDB（中国历代人物传记数据库）仅用于实体规范化，将别名映射到标准名，不参与识别决策。原文句子保持不变，只在输出字段中提供规范名，既保留原始证据又方便后续分析。

B. **实现方案**：优先使用内置别名映射，可选加载CBDB数据库扩充映射表，规范化后的实体列表去重输出。

### 2.3 上下文窗口机制

为解决跨句实体关联问题，设计了一个轻量级的上下文机制：维护最近若干句的高置信度实体列表，仅当实体置信度达到阈值时才加入上下文，避免低质量实体污染。每个传记独立清空上下文，防止不同传记之间的人名串扰。

### 2.4 数据集规模过大与tokens不足问题的解决

A. **问题描述**：按句子切分后得到几十万条数据，如果每条都调用大模型进行评价信息抽取，免费额度很快耗尽。

B. **解决步骤**：

（1）先做NER主体识别，只保留包含人物的段落，再用古文模型筛选出可能包含评价内容的段落，将数据量压缩。

（2）改用Embedding小模型对文本进行编码和向量化，替代直接调用大模型抽取。Embedding模型调用成本低且速度快。

（3）基于向量化结果使用k-means算法进行聚类，实现两类聚合：同一人物在不同段落中评价信息的合并，以及具有相同评价特征的人物集合归类。

### 2.5 智能问答模块优化

A. **用户需求**：问答需要返回原文出处，并进行总结性回答。

B. **技术实现**：采用RAG架构，将抽取出的评价信息向量化后存入向量索引。对用户问题进行向量检索，召回相关段落。将召回段落与用户问题一起输入开源模型生成带出处的回答。

## 三、遇到的问题及解决方法

| 问题 | 解决方法 |
|------|----------|
| 多层级规则投票冲突，识别准确率低 | 重构为微调NER模型主导的单层架构 |
| 大模型tokens不够用 | NER部分改用微调小模型，Embedding用轻量模型 |
| 古文NER效果差 | 用导师提供的数据对GujiRoBERTa进行LoRA微调 |
| 实体置信度计算不合理 | 改用token最小概率策略 |
| 实体边界识别错位 | 修复字符级对齐逻辑，将token标签准确映射到每个字符位置 |
| 问答回答没有出处 | 采用RAG架构，先检索后生成 |
| 句子级数据量过大 | 先用NER筛选，只保留有人物评价的段落 |
| 跨句实体关联问题 | 实现上下文窗口机制，维护最近的高置信度实体列表 |

## 四、实践总结

本次实践周的核心工作是重构毕设中的人物主体识别模块。原有系统采用多层级规则投票机制，存在规则冲突、泛化能力差等问题。根据导师建议，将其改为微调NER模型主导的单层架构，使用导师提供的标注数据进行LoRA微调。

在置信度计算方面，最初采用token概率的平均值，但分析发现一个实体中只要有一个token识别错误就会导致整体错误，因此改用token最小概率策略，使置信度更能真实反映模型可靠性。

数据量控制是关键瓶颈。初始几十万条句子若全部调用大模型，免费额度远不足以支撑。解决方案是先进行NER筛选出包含人物的段落，将数据量压缩后改用Embedding小模型做向量化处理，有效解决了tokens不足问题。

技术实现上，代码中实现了上下文窗口机制维护最近两句的高置信度实体列表，并修复了token到字符的对齐逻辑，使实体边界识别更加准确。智能问答模块采用RAG架构，通过检索召回相关段落再由生成模型输出带出处的回答。

专业学习方面，系统阅读了LoRA微调、Embedding模型及RAG架构的相关资料，这些知识直接指导了毕设的技术实现。本次实践周以自主推进为主，与导师的两次谈话提供了重要方向指引。后续将围绕模型优化和系统完善继续推进。
